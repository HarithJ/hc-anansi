So... What are cron jobs anyway?

Fun fact: The name cron comes from the word "chronos", Greek for "time".
In the context of Unix based systems, a Cron is a time-based job scheduler. Amm...

Simply put, there comes a time where you as user (or more certainly a systems administrator) may want to run certain jobs on a at a partictular time and have the same job(s) recur on a prespecified schedule.

Say for instance you have written a script to automatically run backups of your local files (or that of the users/clients whose systems you administrate over) to a shared network storage: and yes it is the right thing to do. That is itself the definition of a Cron job.

Below is a video to help you better understand what cron jobs are and how you can use Cron and Crontab to create and schedule cron jobs.

Create check

Creating a check is quite straight forward as well. In the following video, notice how one can set an espected period which they (you) expect their job to run. There is also a slider for setting a grace period. The grace period implies that the job has not yet ran in the set timeout but is expected to run after some duration. Simply put, due a number of factors, the job might be late to register the check ergo the need for a grace period before you are notified the check gone down.
